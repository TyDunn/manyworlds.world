TL;DR: The Internet wants the world to remain something we make and could just as easily make differently and become a world where many worlds may fit. For this end, it is beginning to refactor the architecture to better support adaptive software, dynamic data, and a higher velocity of accurate data changes more generally. This is an early draft, so please ask before sharing with others.

Current

Software does not just eat the world; software first creates a mirror of the world and then we restructure ourselves until it fits. Despite the many layers of abstraction, all software development seems to ultimately contribute towards creating a more convincing reflection of either ourselves (from user_id to AI) or the reality that surrounds us (from object_id to VR).

When we believe a part of the mirror fits our desired selves and reality, we press play on that software program and something people want done is automated, merging the mirror and reality into one. If we don't believe it fits, we adjust the reflection or reorganize ourselves until it is acceptable and then press play again, automating and merging again.

Software is a network of networks through which information flows, which is written in code but still often connected together by us. It is frequently referred to as the Internet—the human collective (i.e. we don’t use the network; we are the network). The Internet collectively wants to create better mirrors because it allows us to run parts of reality with less effort.

The mirror is a function of the data we make accessible, which occurs when people or objects generate data that is captured, processed, and stored, ready to be queried and used. When we try to automate parts of reality that are static, software works well. However, most of reality that matters is dynamic, and current software struggles to automate this.

Software today cannot keep up with us as we navigate perpetually novel environments, changing our behavior to improve performance and enhance our chance of survival. It is not capable of supporting a high enough velocity of accurate data changes, which is necessary to create better models of the more dynamic parts of reality that we want to reflect.

After seeing first hand how many of the biggest organizations are attempting to evolve their approach to creating software as well as my experiences as a user of services from Google, Apple, Facebook, Amazon, and many others, it is now clear to me that we are recieving diminishing returns on our efforts to further optimize the current architecture for a higher velocity.

Future

If we want better models of the dynamic world, we need an architecture that enables a higher velocity of accurate data changes on the network. After feeling the pain caused by the current architecture, the Internet has come up with ideas for how to refactor core parts of the architecture and started to work on them in an attempt to further accelerate the velocity.

Rather than building more platforms on top of TCP/IP, HTTP, etc. (i.e. networks operated by a central authrority), we are building more protocols (i.e. networks operated by themselves). This is now possible because tokens can be used as a native primitive of networks, which can align incentives among network participants and make networks self-sustaining.

The appetite of the Internet for a higher velocity of accurate data changes is driving demand for networks operated by themselves because central authorities are the primary bottleneck. This shift pushes data control further towards the edge of the network, where data about users and their context is better captured, processed, and stored.

Right now, data generated by a user or object is separated across disconnected databases for each central authority. Using their collection of narrow snapshots taken from us, central authorities create frequently incorrect and limited reflections. It’s like a mirror of only partial, stiched together images from the past—it leads to undesirable models of ourselves and reality.

Instead, data is starting to be collected together for each user or object and updated as we evolve as much as possible (i.e. dynamic data). That is, this data is captured, processed, and stored with the user or object across all applications. This will empower us to create more desirable mirrors that reflect a sort of live video stream of ourselves and our context.

This better architecture will set us up to create more adaptive software, which is more aware of changes to users and can use that to improve its behavior. Swiftly updating state changes enable us to have a stream that better represents the past, current, and future that surrounds a piece of information (i.e. a high velocity of accurate data changes).

Beyond

Like a techno rave DJ and audience, there is an interplay between the Internet and us. It adapts to us as we steer it, and we adapt to it as it steers us. More adaptive software built using dynamic data, which is made possible by a higher velocity of accurate data changes, can help us increasingly steer our past, present, and future individually and collectively.

This can improve our power to collectively shape and reshape the world, so we don't end up with a single monoculture or a singular truth. It can also empower individuals to shape and reshape their world, so many worlds can exist. Through this, we can better protect and enable possibility, though more problems await us, many that can’t be addressed with code changes.

P.S. The metaphor of the mirror should make it clear why we cannot leave people nor parts of reality behind. We need to ensure people can connect to the Internet and use it responsibly. We need to create a world we all want to inhabit. And we need the humility to accept that the Internet will always need to evolve, as a living entity, as we evolve as living entities.
